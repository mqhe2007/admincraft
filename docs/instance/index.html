# admincraft 实例

.admincraft会导出一个包含了router，store的 Vue 实例，此实例被挂载到页面元素上后就是一个完整的 vue 应用程序。一个应用只包含一个 Admincraft 实例。也正是 admincraft 的完整性保障了模块化开发的可能性，模块工程一旦导入并使用了 admincraft 实例来创建应用，便拥有了主工程的所有能力。

# 实例方法 API

# $addComponent

# 参数：

{String} position 组件的添加位置

{String} name 组件的名称，具有唯一性。

{Object} component 组件构造对象

向应用预设的组件插槽中添加动态组件，具体position取值可查看组件插槽位置列表。

# $destroyComponent

# 参数：

{String} position 组件的添加位置

{String} name 组件的名称，具有唯一性。

卸载已添加的动态组件。

# $addMenus

# 参数：

{Array} 菜单描述对象的数组或者路由配置对象的数组

# 路由配置对象(推荐)

一个符合routes选项要求的数组。

# 菜单对象
[{
  title: '404', // 菜单标题
  name: 'error404', // 路由名称
  index: 0, // 菜单排序
  url: '', // 外链地址
  icon: '', // 菜单图标
  children: [...]
}]
